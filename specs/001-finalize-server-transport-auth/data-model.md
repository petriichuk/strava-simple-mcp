# Data Model: Finalize MCP Server Transport & Auth

**Feature**: `001-finalize-server-transport-auth`
**Date**: 2026-05-31
**Scope note**: This feature does NOT introduce new persisted entities. The
existing modules `src/strava.py`, `src/cache.py`, and `src/metrics.py` are
reused as-is per constitution Principle VI. This document captures the
**logical** entities the feature touches (some persisted, some runtime-only)
so contracts/, quickstart.md, and tasks.md can refer to them by name.

---

## E1. MCP Tool *(runtime / interface)*

A callable endpoint exposed by `src/server.py` over the MCP streamable-HTTP
transport.

**Fields**:
- `name` — string, unique within the server. One of `list_recent_activities`,
  `get_weekly_summary`, `get_activity_detail`.
- `parameters` — JSON Schema for arguments accepted from the client.
- `response_shape` — JSON-serialisable, compact (per constitution Principle II).

**Validation rules**:
- Response payload size MUST be ≤ 4 KB for a representative input (per SC-002).
- Response MUST be JSON-parseable.
- Response MUST NOT contain raw per-second stream arrays.

**Relationships**:
- `get_weekly_summary` and `list_recent_activities` read from **Activity Cache
  Record** (E4).
- `get_activity_detail` reads from **Activity Metrics Record** (E5) on cache
  hit; on miss, computes from streams and writes a new E5.

**Not stored**: the tool registry is in-process only — derived from the
`@mcp.tool()` decorators.

---

## E2. Path Secret *(configuration / credential)*

A high-entropy random string mounted as the URL prefix that gates access to
the MCP application.

**Fields**:
- `value` — string, opaque, treated as a bearer credential.

**Source**: environment variable `MCP_PATH_SECRET`. Loaded once at process
start; not re-read on signal.

**Validation rules** (enforced at startup, per FR-005):
- MUST be set.
- MUST be non-empty.
- MUST be ≥ 24 characters.

**Lifecycle**:
- Generated by the operator (e.g., `python -c "import secrets; print(secrets.token_urlsafe(32))"`).
- Rotation requires: regenerating the value, restarting the server,
  re-registering the connector URL in claude.ai.

**Not stored**: never persisted to SQLite, never written to logs.

---

## E3. Strava Credentials *(persisted in SQLite `tokens` table)*

The OAuth state needed to call the Strava API. Schema already exists in
`src/cache.py`; this feature does not modify it.

**Fields** (per `cache.py` `tokens` schema):
- `id` — always `1` (single-row table).
- `access_token` — short-lived bearer for Strava API calls.
- `refresh_token` — long-lived; rotates on every refresh per Strava policy.
- `expires_at` — epoch seconds at which `access_token` becomes invalid.

**Validation rules**:
- On startup, if the `tokens` row is missing, the `STRAVA_REFRESH_TOKEN` env
  var seeds it via `cache.save_tokens(...)`.
- On refresh, the new `refresh_token` from Strava MUST be persisted (FR-009);
  failure to persist breaks the next startup.

**Lifecycle**: managed by `src/strava.py`'s `StravaClient` (existing code).
This feature exercises but does not modify token handling.

---

## E4. Activity Cache Record *(persisted in SQLite `activities` table)*

A compact summary of one Strava activity. Schema already exists in
`src/cache.py`; not modified.

**Fields** (per `cache.py` `activities` schema):
- `id` — Strava activity id (primary key).
- `start_date` — ISO local date-time string.
- `type` — activity type ("Run", "Ride", etc.).
- `name` — activity name.
- `distance_m`, `moving_time_s`, `elev_gain_m` — numeric.
- `avg_speed_ms`, `avg_hr`, `max_hr` — numeric.
- `iso_year`, `iso_week` — derived at upsert time for cheap weekly grouping.

**Source**: `StravaClient.activities()` paginated fetch after
`last_synced_epoch`.

**Consumer tools**: `list_recent_activities` (returns top-N as a compact JSON
list), `get_weekly_summary` (groups by `iso_year` / `iso_week`).

---

## E5. Activity Metrics Record *(persisted in SQLite `activity_metrics` table)*

The compact computed payload for one activity's detail view. Schema already
exists; not modified.

**Fields** (per `cache.py` `activity_metrics` schema):
- `id` — same Strava activity id as in `activities` (FK).
- `metrics_json` — JSON-encoded payload with these top-level keys:
  - `id`, `name`, `date`, `distance_km`
  - `official_splits_metric` — list of Strava-reported splits, compacted.
  - `stream_metrics` — output of `metrics.compute_stream_metrics(...)`:
    `total_distance_km`, `first_half`, `second_half`, `hr_drift_pct`,
    `decoupling_pct`, `splits`.
  - `time_in_zone` — output of `metrics.summarise_zones(...)`.

**Source**: computed on first read by `get_activity_detail` from streams +
`/zones`, then cached.

**Validation rules**:
- `metrics_json` MUST NOT contain raw stream arrays (the contract of
  `metrics.py` already enforces this — the helper functions reduce streams to
  aggregates before returning).
- Total `metrics_json` size for a 1-hour run SHOULD fit within 4 KB (SC-002).

---

## E6. Sync State *(persisted in SQLite `sync_state` table)*

A small key/value table; the only key this feature touches is
`last_synced_epoch`.

**Fields**:
- `k` — key (text PK).
- `v` — value (text).

**Lifecycle**: `last_synced_epoch` is updated by `_sync_new()` in `server.py`
after every successful pull. Not modified by this feature.

---

## Cross-entity invariants

- An **E4** row exists for every **E5** row (E5 is computed-from-detail and
  detail is only fetched for activities we've seen in the list).
- **E3** is the only entity holding secrets; **E2** is the only credential
  held outside SQLite (in process memory via env).
- The MCP transport layer (E1's host) MUST NOT log either E2 or E3 values.
